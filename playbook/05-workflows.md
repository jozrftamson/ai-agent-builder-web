# Workflows

## Workflow: Neue Produktidee strukturieren

1. Idee in einem Absatz beschreiben.
2. Zielgruppe benennen.
3. Hauptproblem formulieren.
4. Agent-Rolle oder Modul zuordnen.
5. moegliche Inputs und Outputs festhalten.
6. offene Risiken notieren.
7. daraus eine Skill-Idee oder Feature-Idee ableiten.

## Workflow: Neuer Tool-Kandidat

1. Tool benennen.
2. Problem beschreiben, das es loesen soll.
3. Alternativen notieren.
4. Einstiegskosten und Komplexitaet abschaetzen.
5. entscheiden: jetzt nutzen, spaeter pruefen oder verwerfen.
6. Erkenntnisse in `02-tools-uebersicht.md` einpflegen.

## Workflow: Lokalen Skill entwerfen

1. beobachteten Ablauf identifizieren.
2. Trigger und Ziel formulieren.
3. Inputs und Voraussetzungen notieren.
4. konkrete Schrittfolge aufschreiben.
5. Outputs definieren.
6. typische Fehler und Grenzen dokumentieren.
7. Status im Backlog setzen.

## Workflow: Skill-ready pruefen

1. Wurde der Ablauf mehrfach genutzt?
2. Ist der Mehrwert reproduzierbar?
3. Sind Inputs, Schritte und Outputs eindeutig?
4. Gibt es bekannte Pitfalls?
5. Ist der Skill kompakt genug?
6. Wenn ja: in echte Skill-Form ueberfuehren.

## Workflow: Agent-Modul fuer Website planen

1. Rolle benennen.
2. primaeren Nutzen formulieren.
3. Beispielaufgaben sammeln.
4. benoetigte Tools bestimmen.
5. UI-Flaechen und Eingaben skizzieren.
6. erwartete Ergebnisansicht definieren.
7. MVP vs. spaetere Ausbaustufen trennen.
